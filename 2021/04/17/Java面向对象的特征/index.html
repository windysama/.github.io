面向对象的特征

面向对象的特征有那些方面？

1.抽象：

抽象是将一类对象的共同特征总结出来构建类的过程，包括数据抽象和行为抽象两方面。抽象只关注对象有那些属性和i行为，并不关心这些行为的细节是什么

2.继承：

继承是从已有的类得到继承信息创建新类的过程。提供信息的类被成为父类（超类，基类）得到继承信息的类成为子类（派生类）继承让变化中的软件系统有了一定的延续性，同时继承也封装程序中可变因素的重要手段

3.封装：

通常认为封装是把数据和操作数据的方法绑定起来，对数据的访问只能通过已定义的接口。面向对象的本质就是将现实世界描绘成一些列完全自治、封闭的对象。我我们在类中编写的方法就是对实现细节的一种封装。可以说封装就是隐藏一切可以隐藏的东西，只向外界提供最简单的编程接口。

4.多态性

多态性是指运行不同子类的对象对同一消息做出的不同相应。简单说就是用同样的方法但是做了不同的事情。多态分为编译时的多态性和运行时的多态性。如果将对象的方法是为对象想外界提供的服务，那么运行时多态可以解释为：当A系统访问B系统提供的服务时，B系统有多种提供服务的方式，但一切对A系统来说都是透明的。

方法重载实现的是编译时的多态性（也成为前绑定）。运行时的多态是面向对象最精髓的东西，要是实现多态需要做两件事情：1）方法重写，子类继承父类并重写父类中已有的或者抽象的方法）；2）对象造型，用父类型引用子类类型对象，这样同样的引用调用同样的方法就会根据子类对象的不同而表现出不同的的行为
